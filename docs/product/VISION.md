# Vision

## この課題は何か
Dining Philosophers Problem（食事する哲学者問題）のシミュレーション実装。
スレッドとmutexを使用した並行プログラミングの基礎を学ぶ。

## なぜこの課題が存在するか
- マルチスレッドプログラミングの基礎を習得
- 競合状態（race condition）とデッドロックの理解
- mutex による排他制御の実践
- 並行処理における時間管理の重要性を体験

## 完了時の自分の状態
- [ ] pthread_create/join/detach でスレッドを作成・管理できる
- [ ] mutex でリソースの排他制御ができる
- [ ] デッドロックを回避する設計ができる
- [ ] データレースの原因と対策を説明できる
- [ ] gettimeofday でミリ秒単位の時間管理ができる
- [ ] 並行処理のデバッグ手法を知っている

## スコープ外
- ボーナス（プロセス + セマフォ）は mandatory 完了後に検討
- 過度な最適化（まずは正しく動くことを優先）

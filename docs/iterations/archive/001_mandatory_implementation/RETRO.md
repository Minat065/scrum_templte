## 学び
- **mutexは「鍵」であり「金庫」ではない** - 変数を自動で保護するのではなく、プログラマが約束を守る必要がある
- AIDDでも「いきなりコード」ではなく、設計ドキュメントを先に作る方が効率的
- 後から説明するより、先に設計を見せる方が理解が早い

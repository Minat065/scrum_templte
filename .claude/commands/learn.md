# 学習記録

## 使い方
```
/learn
```

## 手順

### 1. 気づきの整理
ここまでで実装したコードの内容を以下の形式で整理
```markdown
## YYYY-MM-DD: [タイトル]

### 何を学んだか
[具体的な内容]

### どういう文脈で
[何をしていて気づいたか]

### 以前の理解
[前はどう思っていたか]

### 新しい理解
[今はどう理解しているか]

### 関連
- [コード、ドキュメント、外部リソースへの参照]
```

### 2. 適切なドキュメントに追記

| 種類 | 追記先 |
|------|--------|
| 技術的な気づき | docs/learning/LEARNINGS.md |
| 課題の意図に関する気づき | docs/learning/INTENT.md |
| 試験で聞かれそうなこと | docs/learning/EXAM_PREP.md |
| スクラムに関する気づき | docs/learning/SCRUM_NOTES.md |

### 3. 確認
追記した内容を報告

## 注意
- 些細なことでも記録
- 「以前の理解」と「新しい理解」の対比を明確に

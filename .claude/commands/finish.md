# 仕上げモード

レビュー（試験）に向けた最終準備。

## 手順

### 1. 現状確認
- 全タスクの完了状況
- ドキュメントの整備状況
- 未解決の課題

### 2. EXAM_PREP.md 完成
以下を整備：
- 必須暗記事項（関数シグネチャ、定義）
- 想定質問と回答
- 弱点・重点復習項目

### 3. 想定質問シミュレーション
以下の観点で質問を生成：

**概念系**
- 使用している技術の基礎概念
- なぜその技術を使うのか

**実装系**
- なぜこの設計にしたか
- 他の選択肢は検討したか
- トレードオフは何か

**エッジケース系**
- 〇〇の場合どうなるか
- エラーハンドリングはどうしているか

### 4. コード説明準備
各ファイル、各関数について：
- 何をしているか
- なぜそうしているか
- 他の方法との比較

### 5. ドキュメント最終確認
- ARCHITECTURE.md は最新か
- DECISIONS.md は網羅的か
- LEARNINGS.md を振り返り
